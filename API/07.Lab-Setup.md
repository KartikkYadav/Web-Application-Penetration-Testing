# Installation (Kali / Debian)

```
apt update
```

```
apt install docker.io docker-compose -y
```

```
systemctl enable docker
```

```
systemctl start docker
```

```
cd /opt
```

```
curl -L -o /tmp/crapi.zip https://github.com/OWASP/crAPI/archive/refs/heads/main.zip
```

```
unzip /tmp/crapi.zip
```

```
cd crAPI-main/deploy/docker
```

```
docker-compose pull
```

```
LISTEN_IP="0.0.0.0" docker-compose -f docker-compose.yml --compatibility up -d
```

# Architecture Breakdown (Based on docker-compose)

# Services Overview

| Service | Purpose |
|------|------|
| crapi-web | Frontend reverse proxy |
| crapi-identity | Authentication + JWT |
| crapi-community | User/community features |
| crapi-workshop | Business logic APIs |
| crapi-gateway | API gateway |

---

# Why crAPI is Important for Pentesting

crAPI was created by **OWASP** to simulate real-world API vulnerabilities.

It helps security testers practice:

- API authorization flaws
- Authentication bypass
- Business logic abuse
- API rate limit bypass
- Object level authorization issues

---

# Pentesting Skills You Can Practice in crAPI

- **BOLA (Broken Object Level Authorization)**
- **Broken Authentication**
- **Mass Assignment**
- **JWT manipulation**
- **Rate limit bypass**
- **Business logic abuse**
- **IDOR testing**

# Example API Testing Workflow

## 1. Capture request in Burp

```
GET /api/users/123
```

## 2. Modify Object ID

```
GET /api/users/124
```

If another user's data is returned → **BOLA vulnerability**

# Best Practice for Learning API Pentesting

1. Run crAPI locally
2. Intercept requests using Burp Suite
3. Map all API endpoints
4. Identify authentication mechanisms
